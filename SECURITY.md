# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 1.0.x   | Yes                |
| < 1.0   | No                 |

## Reporting a Vulnerability

If you discover a security vulnerability in Art in Heaven, please report it responsibly. **Do not open a public issue.**

### Preferred: GitHub Security Advisories

1. Go to the [Security Advisories](https://github.com/stgeorgejc/art-in-heaven/security/advisories) page.
2. Click **"Report a vulnerability"**.
3. Fill in the details and submit.

This allows us to discuss and fix the issue privately before any public disclosure.

### Alternative: Email

Send details to **admin@stshenoudajc.org** with the subject line: `[SECURITY] Art in Heaven`.

Include:
- Description of the vulnerability
- Steps to reproduce
- Affected version(s)
- Any potential impact

### What to Expect

- **Acknowledgment** within 48 hours.
- **Status update** within 7 days with our assessment and timeline.
- **Fix and disclosure** coordinated with you before any public announcement.

We appreciate responsible disclosure and will credit reporters (unless anonymity is preferred).
